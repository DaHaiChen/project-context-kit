# 项目交付文档目录

本目录保存项目级完整交付文档。以后初始化项目上下文后，所有 PRD、需求共识、设计方案、接口设计、QA 待确认、验收记录和复盘文档，优先落在根目录 `docs/` 下。

`.project-context/` 不替代本目录。二者分工如下：

| 目录 | 定位 | 主要读者 | 内容 |
|---|---|---|---|
| `docs/` | 完整交付文档层 | 产品、研发、测试、评审人员、agent | PRD、一期设计、接口设计、规则专题、QA、验收、版本计划 |
| `.project-context/` | 项目语义索引与契约层 | agent、新人、跨模块协作者 | 模块边界、跨模块契约、长期约束、已知坑点、架构决策、代码到文档映射 |

## 推荐目录结构

```text
docs/
  README.md
  文档生成流程.md
  版本计划/
  业务域/
    大模块/
      需求共识.md
      大模块PRD.md
      大模块一期设计.md
      子模块A.md
      子模块B.md
      接口设计.md
      QA待确认.md
      验收记录.md
```

## 落地规则

- 大模块完整需求写在 `docs/<业务域>/<模块>/<模块>PRD.md`。
- 子模块差异需求写在 `docs/<业务域>/<模块>/<子模块>.md`。
- 技术实现方案写在 `docs/<业务域>/<模块>/<模块>一期设计.md`。
- 接口、Java 对接、字段约定等完整设计写在模块目录下的接口设计文档。
- 未决问题写在 `QA待确认.md`，确认后必须回写 PRD、设计或契约。
- 阶段完成后可补充 `验收记录.md`、`复盘.md`。
- 只有长期有效、会影响后续协作的摘要、边界和契约，才同步沉淀到 `.project-context/`。
